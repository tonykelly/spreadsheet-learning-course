Topic 2: Automation and Macros

Automating Repetitive Tasks

Macros allow you to record and replay sequences of actions, dramatically improving efficiency for repetitive tasks. While advanced macro programming is beyond this course, basic macro recording is accessible to all users.

When to Use Automation:

  • Repetitive formatting: Applying the same formatting to multiple reports
  • Data processing: Cleaning and transforming data in the same way repeatedly
  • Report generation: Creating standardized reports with updated data
  • Workflow steps: Multi-step processes that follow the same pattern

Step-by-Step: Recording Your First Macro (Excel)

  1. Prepare for macro recording:
    • Plan out the exact steps you want to automate
    • Practice the sequence manually first
    • Ensure your data is in the expected format
  2. Start recording:
    • View tab → Macros → Record Macro
    • Give your macro a descriptive name
    • Assign a keyboard shortcut (optional)
    • Add a description of what the macro does
  3. Perform your actions:
    • Execute each step carefully and deliberately
    • Avoid unnecessary clicks or selections
    • Use relative references when appropriate
  4. Stop recording:
    • View tab → Macros → Stop Recording
    • Test your macro on sample data
    • Edit if necessary using the Visual Basic Editor

Step-by-Step: Creating a Report Formatting Macro

  1. Set up sample report data:
    • Create a basic data table with headers
    • Include numeric data for formatting
    • Leave formatting plain for macro to apply
  2. Record formatting macro:
    • Select header row
    • Apply bold formatting and background color
    • Format number columns as currency
    • Add borders and adjust column widths
    • Add title and current date
  3. Test and refine:
    • Run macro on different data sets
    • Adjust for varying data ranges
    • Add error handling if needed

Google Sheets Automation with Apps Script

Google Sheets uses Google Apps Script (JavaScript-based) for automation instead of VBA macros.

Step-by-Step: Basic Apps Script Automation

  1. Access Apps Script:
    • Extensions menu → Apps Script
    • Create new project
    • Rename project to describe its purpose
  2. Create simple automation:
    function formatReport() {
    var sheet = SpreadsheetApp.getActiveSheet();
    var range = sheet.getRange("A1:E1");
    range.setFontWeight("bold");
    range.setBackground("#4285f4");
    range.setFontColor("white");
    }
  3. Run and test:
    • Save the script
    • Click Run button
    • Grant necessary permissions
    • Verify results in spreadsheet

Macro Security and Best Practices:

  • Test thoroughly: Always test macros on sample data first
  • Backup data: Create backups before running macros on important data
  • Document code: Add comments explaining what each macro does
  • Security settings: Understand macro security implications
  • Version control: Keep track of macro changes and versions

Topic 3: Best Practices and Optimization

Professional Spreadsheet Development

Creating maintainable, efficient, and professional spreadsheets requires following established best practices and optimization techniques.

Structure and Organization

  • Logical layout: Input, calculation, and output sections
  • Consistent naming: Descriptive sheet and range names
  • Documentation: Comments and instructions
  • Version control: Clear versioning and change tracking
  • Backup strategy: Regular saves and archival

Formula and Function Optimization

  • Efficient formulas: Minimize volatile functions
  • Absolute vs relative: Proper cell reference usage
  • Array formulas: Single formula vs multiple calculations
  • Error handling: Graceful error management
  • Calculation settings: Manual vs automatic calculation
Performance Issue Symptoms Solutions Prevention
Slow Calculation Long delays when entering data or changing values Optimize formulas, reduce volatile functions, use manual calculation Plan formula structure, avoid unnecessary complexity
Large File Size Slow to open/save, difficult to share Remove unused formatting, compress images, split into multiple files Clean up regularly, avoid excessive formatting
Formula Errors #REF!, #VALUE!, #N/A errors appearing Use IFERROR, validate data sources, check references Robust error handling, data validation
Maintenance Issues Difficult to update, break easily when modified Simplify structure, add documentation, use named ranges Design for maintainability from the start

Step-by-Step: Spreadsheet Health Check

  1. File size and performance audit:
    • Check file size and calculation speed
    • Identify sheets with excessive data or formatting
    • Remove unused sheets, rows, and columns
    • Compress or remove large embedded objects
  2. Formula optimization review:
    • Identify volatile functions (NOW, TODAY, INDIRECT)
    • Replace inefficient formulas with better alternatives
    • Use array formulas where appropriate
    • Add error handling to critical formulas
  3. Structure and maintainability assessment:
    • Review naming conventions
    • Add missing documentation
    • Simplify overly complex structures
    • Create separate sheets for different purposes
  4. Data integrity validation:
    • Check for broken links or references
    • Validate data ranges and sources
    • Test all formulas with edge cases
    • Verify calculation results manually

Step-by-Step: Creating Template Standards

  1. Design template structure:
    • Create standard layouts for common reports
    • Include instructions and documentation sheets
    • Set up consistent formatting and styles
    • Add data validation and error checking
  2. Build reusable components:
    • Standard calculation modules
    • Chart templates with consistent styling
    • Dashboard layouts
    • Import/export procedures
  3. Document and share:
    • Create user guides for templates
    • Establish version control procedures
    • Train team members on standard practices
    • Gather feedback and iterate

Collaboration Best Practices:

  • Clear ownership: Define who can edit different sections
  • Version control: Use meaningful file names and change tracking
  • Documentation: Explain complex formulas and processes
  • Data validation: Prevent common input errors
  • Backup strategy: Regular backups and recovery procedures
