Structured Formats

Topic 2: File Format Import (CSV, TXT, JSON)

CSV Import Mastery

CSV (Comma-Separated Values) is the most common data exchange format. Mastering CSV import includes handling various delimiters, encodings, and data quality issues.

Format Delimiter Use Cases Common Issues
CSV Comma (,) Universal data exchange, exports from most systems Commas in data, different encodings, date formats
TSV Tab Data with commas in values, Excel default Tab characters in data, alignment issues
PSV Pipe (|) Database exports, data with commas and tabs Less common, may need custom import settings
Fixed Width Column positions Legacy systems, mainframe exports Alignment issues, trailing spaces

Step-by-Step: CSV Import with Custom Settings

  1. Prepare sample CSV data:
    "Name","Department","Salary","Hire Date","Notes"
    "Smith, John","Sales","50,000","2023-01-15","High performer, promoted"
    "Johnson, Alice","IT","65,000","2022-06-01","Technical lead"
    "Wilson, Bob","Marketing","45,000","2023-03-10","Creative director"
  2. Import using Text Import Wizard:
    • Excel: Data tab ‚Üí Get Data ‚Üí From File ‚Üí From Text/CSV
    • Google Sheets: File ‚Üí Import ‚Üí Upload ‚Üí CSV
    • Choose "Delimited" data type
    • Preview shows how data will be parsed
  3. Configure delimiter settings:
    • Select comma as delimiter
    • Choose text qualifier (usually double quotes)
    • Handle consecutive delimiters (treat as one)
  4. Set column data types:
    • Name: Text
    • Department: Text
    • Salary: Number (remove commas)
    • Hire Date: Date (specify format)
    • Notes: Text
  5. Verify import results:
    • Check that commas in names didn't split columns
    • Verify salary numbers are formatted correctly
    • Ensure dates are recognized properly

Step-by-Step: Advanced Text File Import

  1. Create complex text file example:
    John Smith Sales 50000 2023-01-15 High performer
    Alice Johnson IT 65000 2022-06-01 Technical lead
    Bob Wilson Marketing 45000 2023-03-10 Creative
  2. Use fixed-width import:
    • Choose "Fixed width" in import wizard
    • Set column break lines manually
    • Preview shows column alignment
  3. Handle data quality issues:
    • Trim leading/trailing spaces
    • Handle missing values
    • Convert text numbers to actual numbers

JSON Data Import

JSON (JavaScript Object Notation) is increasingly common for API responses and modern data exchanges.

Step-by-Step: JSON Import and Flattening

  1. Sample JSON structure:
    {
    "employees": [
    {
    "id": 1,
    "name": "John Smith",
    "department": {
    "name": "Sales",
    "manager": "Sarah Jones"
    },
    "salary": 50000
    }
    ]
    }
  2. Excel JSON import:
    • Data tab ‚Üí Get Data ‚Üí From File ‚Üí From JSON
    • Use Power Query Editor to flatten nested data
    • Expand records and lists
    • Transform to table format
  3. Google Sheets approach:
    • Use IMPORTJSON add-on or Google Apps Script
    • Convert JSON to CSV format first
    • Import flattened data

Common Import Problems and Solutions:

  • Character encoding issues: Use UTF-8 encoding for international characters
  • Date format confusion: Standardize date formats during import
  • Number formatting: Remove currency symbols and thousands separators
  • Leading zeros lost: Import as text for ID numbers like ZIP codes
  • Large files: Split into smaller chunks or use streaming import

Topic 3: Data Cleaning and Transformation

Essential Data Cleaning Techniques

Raw imported data often requires cleaning and transformation before analysis. This includes removing duplicates, standardizing formats, and handling missing values.

Text Cleaning

  • TRIM(): Remove leading/trailing spaces
  • UPPER/LOWER/PROPER: Standardize case
  • SUBSTITUTE: Replace unwanted characters
  • Text to Columns: Split combined data
  • Find & Replace: Bulk corrections

Data Validation

  • Remove Duplicates: Identify and eliminate copies
  • Data Types: Ensure proper formatting
  • Range Validation: Check for outliers
  • Consistency Checks: Cross-field validation
  • Completeness: Handle missing values

Step-by-Step: Comprehensive Data Cleaning

  1. Set up messy data example:
    • A1: " JOHN SMITH ", B1: "sales", C1: "50,000.00"
    • A2: "alice johnson", B2: "I.T.", C2: "$65000"
    • A3: "Bob Wilson", B3: "marketing ", C3: "45K"
    • A4: "ALICE JOHNSON", B4: "IT", C4: "65,000"
  2. Clean text data:
    • D1: =PROPER(TRIM(A1)) (clean names)
    • E1: =UPPER(TRIM(B1)) (standardize departments)
    • Fix department inconsistencies: "I.T." ‚Üí "IT"
  3. Clean numeric data:
    • Remove currency symbols: SUBSTITUTE(C1,"$","")
    • Handle abbreviations: IF(RIGHT(C1,1)="K",LEFT(C1,LEN(C1)-1)*1000,VALUE(C1))
    • Convert to consistent numbers
  4. Identify and handle duplicates:
    • Use Data ‚Üí Remove Duplicates
    • Or create helper column: COUNTIF($D$1:D1,D1)&COUNTIF($E$1:E1,E1)
    • Flag potential duplicates for manual review
  5. Validate final dataset:
    • Check for remaining inconsistencies
    • Verify data types are correct
    • Ensure no critical data was lost

Step-by-Step: Advanced Data Transformation

  1. Date standardization:
    • Handle multiple date formats in one column
    • Formula: =DATEVALUE(SUBSTITUTE(SUBSTITUTE(A1,"/","-"),".","-"))
    • Use IFERROR to handle parsing failures
  2. Category standardization:
    • Create lookup table for department mappings
    • Sales, sal, SALES ‚Üí Sales
    • IT, I.T., Information Technology ‚Üí IT
    • Use VLOOKUP or nested IF statements
  3. Data enrichment:
    • Add calculated fields (tenure, age categories)
    • Create derived metrics (performance ratios)
    • Add external data (cost of living by location)

Step-by-Step: Power Query for Excel (Advanced)

  1. Access Power Query:
    • Data tab ‚Üí Get Data ‚Üí From Other Sources ‚Üí Blank Query
    • Or from existing data: Data ‚Üí From Table/Range
  2. Apply transformations:
    • Remove columns not needed
    • Change data types automatically
    • Split columns by delimiter
    • Group and aggregate data
    • Merge with other queries
  3. Create reusable process:
    • Save query steps for future use
    • Refresh automatically when source data changes
    • Share query with team members

Data Cleaning Best Practices:

  • Keep original data: Always preserve raw imported data in separate sheet
  • Document changes: Track what transformations were applied
  • Validate results: Check totals and key metrics before/after cleaning
  • Automate when possible: Create reusable cleaning procedures
  • Test with sample: Try cleaning on small dataset first

Topic 4: External Data Connections

Connecting to Live Data Sources

Modern spreadsheet applications can connect directly to databases, web services, and cloud platforms for real-time data access.

Database Connections

Supported formats: SQL Server, MySQL, Oracle, Access

  • Direct SQL query execution
  • Table and view imports
  • Automatic refresh capabilities
  • Parameter-based queries

Web Data Sources

Supported formats: REST APIs, web pages, cloud services

  • API authentication (OAuth, API keys)
  • JSON/XML response parsing
  • Rate limiting and error handling
  • Scheduled data refreshes

Cloud Platforms

Supported platforms: Google Drive, OneDrive, SharePoint

  • Real-time collaboration
  • Automatic synchronization
  • Version control and backup
  • Cross-platform accessibility

Step-by-Step: Web Data Import

  1. Import from web page table:
    • Excel: Data ‚Üí Get Data ‚Üí From Web
    • Enter URL of page with data table
    • Select specific table from preview
    • Transform data if needed
  2. Set up automatic refresh:
    • Right-click data range ‚Üí Refresh
    • Connection Properties ‚Üí Refresh every X minutes
    • Choose refresh options (background, on open)
  3. Handle dynamic content:
    • Some websites require JavaScript rendering
    • Use Power Query for complex parsing
    • Consider API access for reliable data

Step-by-Step: Google Sheets Integration

  1. Import Google Sheets data:
    • Use IMPORTRANGE function
    • =IMPORTRANGE("spreadsheet_url","Sheet1!A1:C10")
    • Grant permission when prompted
  2. Import web data:
    • IMPORTHTML for web page tables
    • =IMPORTHTML("url","table",1)
    • IMPORTXML for structured data
    • IMPORTDATA for CSV URLs
  3. API integration with Google Apps Script:
    • Extensions ‚Üí Apps Script
    • Create custom functions for API calls
    • Handle authentication and rate limiting

Step-by-Step: Database Connection (Excel)

  1. Set up database connection:
    • Data ‚Üí Get Data ‚Üí From Database ‚Üí From SQL Server
    • Enter server name and database
    • Choose authentication method
  2. Select data:
    • Browse tables and views
    • Preview data before import
    • Write custom SQL query if needed
  3. Configure connection:
    • Set refresh schedule
    • Enable background refresh
    • Handle connection failures gracefully

External Connection Considerations:

  • Security: Store credentials securely, use least-privilege access
  • Performance: Large datasets may slow down spreadsheet
  • Reliability: Handle network failures and timeouts
  • Compliance: Ensure data usage complies with regulations
  • Documentation: Document data sources and refresh schedules
